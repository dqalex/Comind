# TeamClaw UI 设计规格文档

> 供设计师参考的完整页面功能清单，包含每个页面的路由、功能点、交互方式和组件结构。

---

## 一、全局布局

### 整体结构

```
┌──────────────────────────────────────────────────┐
│  Sidebar (左)  │          主内容区 (右)            │
│                │  ┌────────────────────────────┐  │
│  Logo          │  │  Header (顶部标题栏)        │  │
│  导航菜单       │  ├────────────────────────────┤  │
│  项目树         │  │                            │  │
│  AI成员状态     │  │        页面内容区            │  │
│                │  │                            │  │
│  [展开/折叠]    │  └────────────────────────────┘  │
│                │              [全局聊天悬浮按钮]    │
└──────────────────────────────────────────────────┘
```

### Sidebar 侧边栏

| 状态 | 宽度 | 内容 |
|------|------|------|
| 展开 | 224px | Logo + "TeamClaw" 文字、导航菜单（11项）、项目树（可展开/折叠/新建/编辑/删除）、AI 成员在线状态、折叠按钮 |
| 折叠 | 52px | Logo 图标、展开按钮、图标导航 |

**导航菜单项**：Dashboard、Tasks、Schedule、Deliveries、Wiki、Members、Agents、Settings

### Header 顶部栏

- 左：页面标题（动态变化）
- 中：项目选择器下拉框（全局筛选当前项目）
- 右：操作按钮区（每个页面不同）

### 全局聊天系统（悬浮在所有页面之上）

| 组件 | 说明 |
|------|------|
| 悬浮按钮 | 固定右下角，点击弹出聊天抽屉 |
| 聊天抽屉 | 从右侧滑出 |
| 会话管理 | 创建/切换/删除多个对话会话 |
| AI 成员选择 | 选择对话的 AI 对象 |
| 消息流 | 流式 AI 响应（打字机效果）、Markdown 渲染、附件支持 |
| 操作 | Abort 中止、Focus 放大全屏模式 |

### 主题 & 国际化

- Light / Dark 双主题，CSS 变量驱动
- 中文 / English 双语切换

## 二、页面清单（12 个功能页面）

---

### 1. Dashboard 仪表盘

| 属性 | 内容 |
|------|------|
| 路由 | `/dashboard` |
| 定位 | 全局概览 + Gateway 连接管理入口 |

**统计概览（4 张卡片）**：进行中任务数、审核中任务数、待审核交付物数、工作中 AI 成员数

**Gateway 连接管理**：WebSocket 地址输入 + Token 输入 + 连接/断开按钮 + 状态指示灯（绿=已连接/灰=未连接）+ 两种模式（server_proxy / browser_direct）+ Channel 列表

**快捷导航**：6-8 个入口卡片，点击跳转至各功能页

**系统快照（需 Gateway）**：CPU/内存、运行时间、Agent/Session/Cron 数量

---

### 2. Tasks 任务管理 ⭐ 核心页面

| 属性 | 内容 |
|------|------|
| 路由 | `/tasks` |
| 定位 | AI 协作任务看板，项目核心功能 |

**工具栏**：视图切换（Board/List）、新建任务、筛选（优先级/来源）、多选模式、批量推送 AI

**看板视图（Board）**
- 4 列：待处理(todo) → 进行中(in_progress) → 审核中(reviewing) → 已完成(completed)
- 按项目分泳道（泳道可折叠/展开）
- 任务卡片：标题、优先级色标、指派人头像、来源标记
- **拖拽**：跨列拖拽改状态、跨泳道拖拽改项目
- 每列底部快速创建输入框

**列表视图（List）**：按项目分组的表格，列含状态/标题/优先级/指派人/时间

**新建任务弹窗**：标题(必填)、描述(Markdown)、项目、优先级(high/medium/low)、指派人

**任务详情抽屉（从右侧滑出，3 个 Tab）**

| Tab | 功能 |
|-----|------|
| 详情 | 编辑标题/描述(自动保存)、状态/优先级/指派人、附件文档选择器 |
| 检查清单 | Checklist 子项增删改、完成进度条 |
| 活动 | 评论 + 操作日志(自动记录状态变更) |

- "推送到 AI" 按钮：将任务发送给 AI 成员处理

---

### 3. Wiki 知识库

| 属性 | 内容 |
|------|------|
| 路由 | `/wiki` |
| 定位 | AI 可读的团队知识库 |
| 布局 | 左侧文档列表(1/3) + 右侧编辑区(2/3) Master-Detail |

**左侧文档列表**
- 搜索框（按标题，debounce）
- 类型过滤下拉
- 新建文档按钮
- 按类型分组折叠：guide/reference/report/note/decision/scheduled_task/task_list/other
- 文档项：标题 + 类型图标 + 更新时间

**右侧编辑区**
- 文档标题内联编辑
- 类型/项目标签选择
- Markdown 编辑器（实时编辑 + 自动保存）
- 知识图谱：双向链接 `[[文档标题]]`、反向链接、关联项目/@提及人员/关联任务
- 分享（复制链接）、标签编辑器

**新建文档弹窗**：标题、类型、项目关联、内置模板可选

**URL 参数**：`?doc=xxx` 自动定位到指定文档

---

### 4. Members 成员管理

| 属性 | 内容 |
|------|------|
| 路由 | `/members` |
| 定位 | 人类 + AI 成员统一管理 |

**AI 成员区**：卡片列表（名称/角色/在线状态/Session 数）、Gateway Agent 健康状态、查看/生成/复制 API Token、**一键配置(Quick Setup)**：自动为 AI 创建专属项目和初始任务

**人类成员区**：卡片列表、添加/编辑/删除

---

### 5. Deliveries 交付物审核

| 属性 | 内容 |
|------|------|
| 路由 | `/deliveries` |
| 定位 | AI 产出物的审核流程 |

**统计卡片（4 个）**：全部、待审核、已通过、需修改/退回

**状态过滤标签**：all / pending / approved / rejected / revision_needed

**交付物列表**：按项目分泳道（可折叠），卡片含标题/状态/来源平台(腾讯文档/飞书/Notion/本地)/关联任务/提交时间

**审核弹窗**：通过/退回/需修改 + 审核评论

---

### 6. Projects 项目管理

| 属性 | 内容 |
|------|------|
| 路由 | `/projects` |
| 定位 | 项目 CRUD + 自动巡检配置 |

**项目卡片列表**：名称、描述、任务数/已完成数/文档数统计

**来源过滤**：all / local / openclaw

**项目 CRUD**：新建弹窗、内联编辑名称/描述、删除(确认)

**自动巡检配置弹窗**：为项目设置定时巡检 Cron Job，选择 Agent + 巡检间隔(30min~24h)，已配置巡检的项目显示 Job 状态

---

### 7. Agents 管理

| 属性 | 内容 |
|------|------|
| 路由 | `/agents` |
| 定位 | AI Agent 生命周期管理（需 Gateway 连接） |
| 布局 | 左侧 Agent 列表侧栏 + 右侧 7-Tab 详情面板（Master-Detail） |

**左侧 Agent 列表**：健康状态、Session 数、默认标记、创建/删除

**右侧 7 个 Tab**

| Tab | 功能 |
|-----|------|
| Overview | Agent 基本信息、配置表单(加载/保存/重载)、健康状态 |
| Files | Agent 文件管理 |
| Tools | Agent 工具列表管理 |
| Skills | Agent 技能启用/禁用 |
| Channels | Agent 频道配置 |
| Cron | Agent 定时任务(启用/禁用/手动运行/删除) |
| Sessions | Agent 会话列表 |

---

### 8. Schedule 定时任务调度

| 属性 | 内容 |
|------|------|
| 路由 | `/schedule` |
| 定位 | Cron Job 全生命周期管理（需 Gateway） |

**创建表单弹窗**：名称、Agent 选择、调度方式(every 间隔 / cron 表达式)、时区、Session 目标(main/isolated)、唤醒模式(now/next-heartbeat)、Payload 类型(agentTurn/systemEvent)、交付模式(announce/webhook/none)

**任务列表**：按启用/禁用分组，启用/禁用切换、手动运行、编辑、删除

**运行历史**：展开查看每个 Job 的运行记录(ok/error/skipped)、下次唤醒时间

---

### 9. Settings 系统设置

| 属性 | 内容 |
|------|------|
| 路由 | `/settings` |
| 布局 | 5 个 Tab 页签 |

| Tab | 功能 |
|-----|------|
| 通用(General) | 主题切换(Light/Dark)、语言切换(中/英)、数据刷新 |
| OpenClaw | Gateway 连接模式配置、Workspace 管理(添加/刷新)、远程连接指南 |
| 安全(Security) | SSRF 防护、外网访问开关(风险确认)、DNS 重绑定防护、私有 IP 控制 |
| 调试(Debug) | DebugPanel 调试工具 |
| 关于(About) | 版本号、技术栈 |

**URL 参数**：`?tab=xxx` 自动切换 Tab

---

### 10. Settings/OpenClaw 专属设置

| 属性 | 内容 |
|------|------|
| 路由 | `/settings/openclaw` |
| 定位 | OpenClaw 独立设置子页面 |

Gateway 配置面板 + Workspace 列表管理（列表/新建/刷新）+ 返回 Settings

## 三、交互模式汇总

| 交互模式 | 使用页面 | 说明 |
|----------|---------|------|
| 拖拽(Drag & Drop) | Tasks | 跨状态列/跨项目泳道拖拽任务卡片 |
| 看板(Kanban) | Tasks | 4 列 + 项目泳道 |
| 泳道分组(Swimlane) | Tasks、Deliveries | 按项目分组，可折叠 |
| Master-Detail | Agents、Wiki | 左列表 + 右详情面板 |
| Tab 切换 | Agents(7)、Settings(5)、TaskDrawer(3) | 标签页内容切换 |
| 弹窗/对话框 | 所有页面 | 新建、编辑、删除确认、审核 |
| 抽屉(Drawer) | Tasks、Chat | 从侧边滑出的详情面板 |
| 搜索过滤 | Sessions、Skills、Wiki、Tasks、Deliveries | 搜索框 + 类型/状态过滤标签 |
| 内联编辑 | Projects、Sessions、Wiki | 点击直接编辑，自动保存 |
| 实时自动保存 | Wiki | Markdown 编辑 debounce 自动保存 |
| SSE 实时推送 | 全局 | DataProvider 监听事件自动刷新数据 |
| 流式聊天 | Chat | 打字机效果的 AI 响应 |
| 双视图切换 | Tasks | Board / List 模式 |
| 多选批量操作 | Tasks | 多选 → 批量推送给 AI |
| 快捷键 | 所有弹窗 | Escape 关闭 |

## 六、设计建议

1. **信息密度**：这是生产力工具，信息密度要高，避免大面积留白
2. **状态可视化**：大量状态标签（任务状态、连接状态、审核状态），建议用色彩+图标双重编码
3. **响应式**：当前仅桌面端，最小宽度 1280px
4. **暗色模式**：所有页面都需要 Dark Mode 适配
5. **AI 感知**：AI 成员和人类成员在视觉上要有明确区分（如紫色光环/标记）
6. **实时感**：连接状态、在线状态用呼吸灯/脉冲动画，聊天用打字机效果
